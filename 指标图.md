## 💡 **为什么要两轮迭代？**

| 轮次 | 目的 | 输入 | 处理方式 | 输出 |
|------|------|------|--------|------|
| **第1轮** | 全面扫描 | 所有指标 | 宽泛分析 | 识别关键领域 |
| **第2轮** | 精确诊断 | 关键指标 | 深度分析 | 精确优先级 |

**类比**：
```
第1轮：医生看全身 → 发现"血液系统可能有问题"
第2轮：医生专注血液 → 发现"WBC 和 RBC 都异常，优先血液科"
```

### **1️⃣ [indicator_graph/indicator_gat.py] ⭐ 核心计算逻辑**

这里才是真正做工作的地方！

```python
class IndicatorGAT:
    def forward(self, patient_labs: Dict[str, float]):
        """完整的推理管道"""
        
        # 【第1步】计算异常程度 Z-score
        abnormality_scores = self.compute_abnormality_scores(patient_labs)
        #   输入：{"WBC": 7.2, "RBC": 4.72, ...}
        #   输出：{"WBC": 0.1, "RBC": 0.2, ...}（0-1 之间）
        #   算法：|患者值 - 参考中值| / 标准差
        
        # 【第2步】GAT 注意力权重计算（邻域传播）
        attention_weights = self.compute_attention_weights(patient_labs)
        #   不仅考虑自身异常程度
        #   还考虑在生理关联图上，邻域指标的异常程度
        #   例如：WBC 异常 → 看它相关的 RBC、HCT 也异常吗？
        #   K=2 次迭代
        
        # 【第3步】识别关键指标簇
        clusters = self.identify_key_clusters(patient_labs, top_k=5)
        #   输出：Top-5 关键异常指标
        #   输出：相关指标的簇（通过 BFS 找邻域）
        
        return {
            'key_indicators': [...],
            'weights': {...},
            'clusters': [...],
            'abnormality_scores': {...}
        }
```

**关键的 3 个方法**：

| 方法 | 行数 | 功能 |
|------|------|------|
| `compute_abnormality_scores()` | L60-94 | 计算Z-score异常程度 |
| `compute_attention_weights()` | L96-155 | **GAT核心**：邻域消息传递 |
| `identify_key_clusters()` | L157-200 | 识别Top-K异常指标簇 |

### **3️⃣ [graph_inference.py] — 中枢管理**

```python
# 全局模型缓存
_indicator_gat = None
_expert_gat = None

def init_graph_models():
    """启动时初始化"""
    global _indicator_gat, _expert_gat
    graph_loader = GraphLoader()
    indicator_graph = graph_loader.load_indicator_graph()  # ← 调用图加载器
    
    # 用图和参考范围初始化 IndicatorGAT
    _indicator_gat = IndicatorGAT(
        indicator_graph=indicator_graph,
        reference_ranges=REFERENCE_RANGES  # ← 来自 reference_ranges.py
    )

def get_graph_models():
    """返回缓存的模型"""
    if _indicator_gat is None:
        init_graph_models()
    return (_indicator_gat, _expert_gat)
```
