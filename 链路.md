### **第三步：Python OCR 服务识别并提取指标**

**[ai-services-python/ocr_service/main.py]** - OCR识别和指标提取核心

```python
# 入口：POST /api/v1/analyze-vision
analyze_vision(request: AnalyzeVisionRequest) {
  1. 查 Redis 缓存：
     cache_key = "ocr_result:{file_path_hash}"
     如果命中 → 直接返回 ✓ (<100ms)

  2. 缓存未命中 → 下载图片：
     image_base64 = _download_image_as_base64(filePath)

  3. 调用 DashScope qwen3-vl-32b-thinking 识别（或Mock模式）：
     llm_analysis = _call_dashscope_vision(image_base64)
     # 返回格式：
     # 白细胞计数(WBC)：7.2 × 10³/μL
     # 红细胞计数(RBC)：4.8 × 10⁶/μL
     # 血红蛋白(HB)：14.2 g/dL
     # ... 等所有指标

  4. 结构化处理 llm_analysis：
     _build_structured_payload(llm_analysis) {
       ├─ 标准化指标名称（_normalize_indicator_key）
       │  例如："白细胞计数" → "wbc"
       ├─ 提取数值（_extract_numeric_value）
       │  例如："7.2" 从 "7.2 × 10³/μL"
       ├─ 构建 patient_labs 字典：
       │  {"wbc": 7.2, "rbc": 4.8, "hemoglobin": 14.2, ...}
       └─ 返回三层结构：
           {
             "analysis": [...],          # 原始文本行
             "full_extraction": [...],   # 完整提取内容
             "gat_structured": {         # ← 关键：供GAT使用
               "patient_labs": {"wbc": 7.2, "rbc": 4.8, ...},
               "mapped_count": 12,  # 成功标准化的指标数
               "coverage": "85%"    # 覆盖率
             }
           }
     }

  5. 写入 Redis 缓存（TTL=7天）

  6. 返回结构化结果
}
```

**关键指标映射表** (在代码中定义)：

- `wbc` → 白细胞、WBC、白血球
- `rbc` → 红细胞、RBC、红血球
- `creatinine` → 肌酐、CREA、Cr
- `bun` → 尿素氮、BUN

### **第六步：GAT 图推理分析指标（关键！）**

**[langchain_service/core/agent_streaming.py]** - \_run_graph_inference_iterative 方法 (L82-131)

```python
_run_graph_inference_iterative(lab_results: {"wbc": 7.2, "rbc": 4.8, ...}) {

  1️⃣ 标准化指标名称（适配图节点）：
     _normalize_labs_for_graph(lab_results)
     # 将 "wbc" 映射为 "WBC"（标准医学符号）
     # 输出：{"WBC": 7.2, "RBC": 4.8, ...}

  2️⃣ 获取 GAT 模型：
     indicator_gat, expert_gat = get_graph_models()

  3️⃣【第一轮迭代】全量指标推理：
     indicator_result = indicator_gat.forward(normalized_labs)
     # indicator_gat 来自 graph/indicator_graph/indicator_gat.py
     # ↓ IndicatorGAT.forward() 内部：
     #   1. compute_abnormality_scores()  → 计算Z-score异常程度
     #   2. compute_attention_weights()   → GAT注意力权重+邻域传播
     #   3. identify_key_clusters()       → 识别Top-K异常指标簇
     #
     # indicator_result = {
     #   'key_indicators': ["WBC", "RBC", "Hb"],  # 关键异常指标
     #   'weights': {"WBC": 0.8, "RBC": 0.6, ...}, # 权重
     #   'clusters': [{"WBC", "RBC", "Hb"}, ...],  # 相关指标簇
     #   'abnormality_scores': {"WBC": 0.7, ...}  # 异常分数
     # }

  4️⃣ ExpertGAT 推荐科室：
     expert_result = expert_gat.forward(key_indicators, weights)
     # expert_result 来自 graph/department_agent_graph/expert_gat.py
     # 返回推荐科室+权重：
     # {
     #   'recommended_agents': ["血液科", "感染科"],
     #   'agent_weights': {"血液科": 0.9, "感染科": 0.6}
     # }

  5️⃣ 生成 Prompt Injection：
     prompt_injection = _generate_prompt_injection(
       key_indicators,
       indicator_result,
       expert_result['recommended_agents'],
       expert_result
     )
     # 返回格式化的提示词段落，供LLM使用
     # 例如：
     # "【图推理第1轮】
     #  关键异常指标: WBC(0.8), RBC(0.6)
     #  推荐科室: 血液科(0.9), 感染科(0.6)
     #  ..."

  6️⃣【第二轮迭代】聚焦关键指标推理：
     focused_labs = {k: v for k, v in 原始labs
                     if k in first_round_key_indicators}
     # 第二轮仅用 Top-5 指标重新运行GAT推理
     # 模拟ReAct"观察后收缩搜索空间"的效果

  ↓
  返回完整的两轮图推理结果（字符串格式）
}
```

**GAT 系统的完整工作流**：

```
patient_labs (Dict[str, float])
    ↓
[IndicatorGAT.compute_abnormality_scores()]
    ↓ 计算每个指标的异常程度 Z-score
    ↓
[IndicatorGAT.compute_attention_weights()]
    ↓ 通过图神经网络传播：
    ├─ 自身贡献 70%
    ├─ 邻域异常指标贡献 30%（K=2迭代）
    ↓
[IndicatorGAT.identify_key_clusters()]
    ↓ 识别 Top-5 关键异常指标簇
    ├─ 聚类：关键指标 + 其邻域指标
    ↓
key_indicators + weights
    ↓
[ExpertGAT.forward()]
    ↓ 推荐科室
    ├─ 血液科、肾内科、肝胆病科、内分泌科、呼吸科
    ↓
expert_result (推荐科室+权重)
    ↓
[_generate_prompt_injection()]
    ↓ 格式化为系统提示词
    ↓
注入到 LLM 上下文 → 引导诊断方向
```
